
**TCP/IP模型**是互联网的核心通信架构，它是一个**实用导向的分层模型**，和OSI的七层理论模型不同，TCP/IP模型更贴合实际网络部署，常见划分是**四层结构**，也有部分资料会将其细化为五层（兼容OSI的分层思想）。

### 一、 TCP/IP四层模型（核心标准）
从底层到顶层依次分工，每层对应不同的协议和功能，且数据在每层会被封装/解封装。

| 层级 | 名称 | 核心功能 | 典型协议 | 对应OSI层级 |
|------|------|----------|----------|-------------|
| 第4层 | **应用层** | 直接为用户应用提供网络服务，定义应用程序间的通信规则 | HTTP、HTTPS、FTP、SMTP、DNS、SSH | 应用层+表示层+会话层 |
| 第3层 | **传输层** | 提供**端到端**的通信服务，实现可靠/高效传输 | TCP、UDP | 传输层 |
| 第2层 | **网际层** | 实现**跨网络的路由与寻址**，负责将数据包从源主机发送到目标主机 | IP（IPv4/IPv6）、ICMP、ARP | 网络层 |
| 第1层 | **网络接口层** | 负责**物理介质上的比特流传输**，处理硬件相关的通信细节 | 以太网、WiFi、PPP、帧中继 | 数据链路层+物理层 |

### 二、 补充：TCP/IP五层模型（教学常用）
为了更直观地和OSI七层模型对应，教学中常把四层模型的**网络接口层**拆分为两层，形成五层结构：
**物链网输用**

1.  **应用层**（对应OSI应用层+表示层+会话层）
2.  **传输层**（对应OSI传输层）
3.  **网络层**（对应OSI网络层）
4.  **数据链路层**（对应OSI数据链路层）
5.  **物理层**（对应OSI物理层）

### 三、 TCP/IP模型与OSI模型的核心区别
| 维度   | TCP/IP模型            | OSI模型             |     |
| ---- | ------------------- | ----------------- | --- |
| 定位   | 实用型架构，互联网的实际标准      | 理论参考模型，用于规范和教学    |     |
| 分层数  | 四层（或五层）             | 七层                |     |
| 设计思路 | 先有协议，后有模型（基于已有协议归纳） | 先有模型，后有协议（基于理论设计） |     |
| 核心协议 | IP、TCP、UDP、HTTP等    | 无特定协议，仅定义功能框架     |     |

### 四、 数据传输的封装/解封装过程（以四层模型为例）
1.  **发送方封装**：应用层数据 → 传输层封装TCP/UDP首部 → 网际层封装IP首部 → 网络接口层封装帧首部/尾部 → 物理层转换为比特流传输。
2.  **接收方解封装**：物理层接收比特流 → 网络接口层解帧 → 网际层解IP包 → 传输层解TCP/UDP段 → 应用层获取原始数据。

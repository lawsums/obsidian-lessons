### 一、Windows 系统（chs.exe 多为Windows相关进程，如输入法进程）

#### 方法2：命令行（CMD/PowerShell，精准查询）
1. 打开命令工具：按 `Win+R` 输入 `cmd`（或 `powershell`），回车打开。
2. 执行查找命令：
   - CMD中输入：`tasklist | findstr chs.exe`，回车后显示 `chs.exe` 的 **映像名称、PID、内存占用**（例：`chs.exe                     1234 Console                    1    123,456 K`）。
   - PowerShell中输入：`Get-Process chs`（无需写.exe，支持模糊匹配），回车后直接显示进程的PID、CPU、内存占用，更清晰。

### 二、Linux/macOS 系统（chs.exe 较少见，多为Windows程序兼容运行或自定义进程）
这类系统中 `.exe` 不是原生进程格式，可能是通过 Wine 运行的Windows程序，查找方法如下：

#### 方法1：命令行（核心方式，必用）
1. 打开终端（Linux：Ctrl+Alt+T；macOS：Launchpad→其他→终端）。
2. 执行以下命令（按需求选，优先前2个）：
   - 精准查找进程（含PID、命令行）：`ps -ef | grep chs.exe | grep -v grep`  
     （解释：`ps -ef` 列所有进程，`grep chs.exe` 过滤目标，`grep -v grep` 排除搜索本身的临时进程）。
   - 直接获取PID：`pgrep -f chs.exe`  
     （解释：`-f` 按完整命令行匹配，避免漏查Wine运行的 `chs.exe`，直接输出PID）。
   - 动态监控：`top | grep chs.exe`（实时刷新 `chs.exe` 的CPU/内存占用，按 `q` 退出）。

# 找到了之后怎么终止进程
查到查到 `chs.exe` 进程后，核心是“先正常终止，失败再强制终止”，分 Windows、Linux/macOS 两种系统，按图形化（直观）和命令行（精准）整理操作，附带注意事项与进阶技巧，避免误操作。

#### 方法1：图形化（任务管理器，最安全）
1. 打开任务管理器（`Ctrl+Shift+Esc`），搜索 `chs.exe` 定位进程。
2. 常规终止：右键 → 结束任务（允许进程保存数据、释放资源）。
3. 强制终止：若提示失败，切换到「详细信息」标签，右键 → 结束进程树（终止进程及子进程）。
4. 权限不足：以管理员身份重新打开任务管理器（任务栏右键 → 以管理员身份运行）。

#### 方法2：命令行（CMD/PowerShell，适合脚本）
先查 PID：`tasklist | findstr chs.exe`（CMD）或 `Get-Process chs`（PowerShell）获取 PID。

| 命令场景       | 命令示例                                                                       | 说明                        |
| :--------- | :------------------------------------------------------------------------- | :------------------------ |
| 按 PID 强制终止 | `taskkill /PID 1234 /F`（CMD）<br>`Stop-Process -Id 1234 -Force`（PowerShell） | /F 强制终止，-Force 等效，适合无响应进程 |
| 按进程名终止所有实例 | `taskkill /IM chs.exe /F`                                                  | 终止所有 chs.exe，慎用（可能影响输入法）  |
| 终止进程树      | `taskkill /PID 1234 /F /T`                                                 | /T 包含子进程，适合有依赖的进程         |

---

### 二、Linux/macOS 系统（chs.exe 多为 Wine 运行，需匹配路径）
先查 PID：`pgrep -f chs.exe`（直接输出 PID）或 `ps -ef | grep chs.exe | grep -v grep`（过滤临时进程）。
#### 方法1：命令行（终端，核心方式）
| 命令         | 示例                                        | 信号说明                        |
| :--------- | :---------------------------------------- | :-------------------------- |
| 正常终止（推荐）   | `kill 1234`                               | 发送 SIGTERM（15），进程可清理资源      |
| 强制终止（无响应时） | `kill -9 1234`                            | 发送 SIGKILL（9），立即终止，可能丢失数据   |
| 按名称终止      | `pkill -f chs.exe` 或 `killall -9 chs.exe` | pkill 按命令行匹配，killall 按进程名匹配 |

#### 方法2：图形化（辅助）
- Linux：系统监视器 → 搜索 `chs.exe` → 选中 → 结束进程（或强制结束）。
- macOS：活动监视器 → 选中 `chs.exe` → 点击左上角「X」→ 选择退出/强制退出。

---

### 三、关键注意事项（必看）
1. 合法性校验：正常 chs.exe 路径为 `C:\Windows\System32\chs.exe`（Windows），异常路径（如桌面、Downloads）可能为恶意程序，建议先查文件属性与数字签名。
2. 系统进程风险：终止系统关键进程（如输入法、服务）可能导致崩溃、功能异常，优先关闭父程序而非直接杀进程。
3. 数据安全：强制终止前，尝试关闭程序窗口或发送正常终止信号（`kill PID` 或不加分隔符的 taskkill），避免数据丢失。
4. 权限问题：Windows 需管理员权限，Linux/macOS 加 `sudo`（如 `sudo kill -9 1234`）终止系统/其他用户进程。

---

### 四、进阶：处理顽固进程（无法终止时）
1. 排查句柄占用：Windows 打开资源监视器（任务管理器 → 性能 → 打开资源监视器），搜索 `chs.exe` 查看句柄，关闭占用程序后再终止。
2. 解锁工具：Windows 用 Process Explorer（Sysinternals）→ 查找 → 句柄/文件 → 输入 `chs.exe` 定位占用，关闭占用进程。
3. 禁用自启动：终止后防止重启，Windows 用 `msconfig` 或任务管理器「启动应用」禁用；Linux 检查 `crontab`、`systemd` 服务；macOS 查「系统设置 → 通用 → 登录项」。

### 五、C++ 编程终止进程（开发者场景）
#### Windows（Toolhelp32 API）
```cpp
#include <tlhelp32.h>
void TerminateProcessByPID(DWORD pid) {
    HANDLE hProcess = OpenProcess(PROCESS_TERMINATE, FALSE, pid);
    if (hProcess) { TerminateProcess(hProcess, 0); CloseHandle(hProcess); }
}
// 调用示例：TerminateProcessByPID(1234);
```
#### Linux/macOS（kill 系统调用）
```cpp
#include <signal.h>
void TerminateProcessByPID(pid_t pid) {
    kill(pid, 9); // SIGKILL 强制终止，15 为 SIGTERM 正常终止
}
// 调用示例：TerminateProcessByPID(1234);
```
